# v1.2 HTTPS with Automatic Certificates - Requirements

## Milestone Overview

**Goal:** Implement HTTPS support con certificados TLS automáticos generados on-the-fly para desarrollo local seguro sin configuración manual.

**Timeline:** 2026-02-22 → 2026-02-23 (Phase 13-14 complete)

**Dependencies:** v1.1 Advanced Protocols shipped

## Success Criteria

The milestone is COMPLETE when:

1. ✅ Certificate Authority (CA) local se genera automáticamente en primera ejecución
2. ✅ Wildcard certificates para `*.localhost` se generan con SAN correctos
3. ✅ CA certificate se instala en Windows Certificate Store (manual en macOS/Linux)
4. ✅ HTTPS endpoint en proxy (puerto 1356, configurable)
5. ✅ Browsers aceptan HTTPS sin warnings (después de trust installation)
6. ✅ Certificate renewal automática antes de expiración
7. ✅ CLI commands para gestión de certificados

## Known Limitations

### By Design (Deferred to v1.3+)
- **macOS/Linux Trust:** Manual installation only (automated trust deferred)
- **HTTPS Proxy:** Not implemented (certificate management complete, proxy integration pending)
- **Certificate Lifecycle:** Background monitoring not implemented (manual checks via CLI)
- **Integration Tests:** Automated tests not written (manual testing performed)
- **Documentation:** User guides not created (code is self-documenting con help text)

### Platform Limitations
- **Windows Only:** Automated trust installation works only on Windows 10+
- **Admin Required:** Certificate trust operations require administrator privileges
- **Store Access:** Windows Certificate Store operations may fail if store is corrupted

### Certificate Limitations
- **Self-Signed:** Certificates are self-signed (not suitable for production)
- **5-Year Validity:** Fixed 5-year expiration (not configurable)
- **Single CA:** Only one CA certificate supported (multiple CAs not supported)

## Migration Path from v1.1

**For Users:**
1. Install v1.2 via `dotnet tool install --global portless.dotnet`
2. Run `portless cert install` to trust CA certificate (Windows only, requires admin)
3. Run `portless cert status` to verify trust status
4. Run `portless cert renew` to regenerate certificate (if within 30 days of expiration)

**Breaking Changes:** None

**Upgrade Path:** Direct upgrade from v1.1 to v1.2 (no configuration changes required)

**Rollback:** Uninstall v1.2, install v1.1 (certificates in `~/.portless/` preserved)
