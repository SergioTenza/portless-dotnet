## Known Limitations

### By Design (Deferred to Future Releases)
- **macOS/Linux Automated Trust:** Manual installation required (documented)
- **Background Monitoring:** Opt-in via PORTLESS_ENABLE_MONITORING=true
- **E2E Testing:** Automated integration tests use TestServer (not real Kestrel)

### Platform Limitations
- **Windows Only:** Automated trust installation works only on Windows 10+
- **Admin Required:** Certificate trust operations require administrator privileges
- **Manual macOS/Linux:** Users must manually install CA certificate on non-Windows platforms

### Certificate Limitations
- **Self-Signed:** Certificates are self-signed (not suitable for production)
- **5-Year Validity:** Fixed expiration (not configurable)
- **Single CA:** Only one CA certificate supported

## Migration Path

**For Users Upgrading from v1.1:**
1. Install v1.2 via `dotnet tool install --global portless.dotnet --version 1.2.0`
2. Run `portless cert install` (Windows only, requires admin)
3. Run `portless cert status` to verify trust
4. Run `portless proxy start --https` to enable HTTPS endpoint

**Rollback:** Uninstall v1.2, install v1.1 (certificates in `~/.portless/` preserved)
