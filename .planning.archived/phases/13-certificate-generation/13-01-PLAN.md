  <action>
    Extend CertificateService with GenerateWildcardCertificateAsync method:

    **GenerateWildcardCertificateAsync method:**
    - Create RSA key with 2048-bit size (from options.ServerKeySize)
    - Create CertificateRequest with X500DistinguishedName("CN=*.localhost")
    - Create SubjectAlternativeNameBuilder and add:
      - AddDnsName("localhost")
      - AddDnsName("*.localhost")
      - AddIpAddress(IPAddress.Loopback) for 127.0.0.1
      - AddIpAddress(IPAddress.IPv6Loopback) for ::1
    - Add SAN extension via request.CertificateExtensions.Add(sanBuilder.Build())
    - Add X509BasicConstraintsExtension(certificateAuthority: false, critical: true)
    - Add X509EnhancedKeyUsageExtension with OID "1.3.6.1.5.5.7.3.1" (Server Authentication)
    - Add X509KeyUsageExtension(DigitalSignature | KeyEncipherment, critical: false)
    - Get CA's private key via caCertificate.GetRSAPrivateKey() - throw InvalidOperationException if null
    - Generate 12-byte random serial number using RandomNumberGenerator.Fill()
    - Ensure serial number is positive: serialNumber[0] &= 0x7F
    - Create certificate via request.Create(caCertificate, notBefore, notAfter, serialNumber)
    - Export and reload with X509KeyStorageFlags.Exportable

    **Validation:**
    - Validate caCertificate.HasPrivateKey before attempting to sign
    - Throw ArgumentException if CA lacks private key with clear error message

    **Key patterns from RESEARCH.md:**
    - Use SubjectAlternativeNameBuilder for DNS and IP SANs
    - Use request.Create(issuer, notBefore, notAfter, serialNumber) for CA-signed certificate
    - Server certificate must have serverAuth EKU (1.3.6.1.5.5.7.3.1)
    - Serial number must be positive (first byte & 0x7F)
