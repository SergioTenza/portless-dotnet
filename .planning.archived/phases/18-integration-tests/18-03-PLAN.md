Create MixedProtocolRoutingTests class:
1. Create test class implementing `IClassFixture<WebApplicationFactory<Program>>`
2. Add XML doc comment: "/// <summary>\n/// Mixed protocol routing integration tests.\n/// Tests verify proxy supports HTTP and HTTPS backends simultaneously with correct SSL validation.\n/// </summary>"
3. Add private fields: `WebApplicationFactory<Program> _factory`, `DynamicConfigProvider _config`
4. Implement constructor injecting dependencies, resolve `_config` from services
5. Create test method `Mixed_Http_And_Https_Backends_Configured_Simultaneously()`:
   - Arrange: Create routes for `http-backend.localhost` and `https-backend.localhost`
   - Configure clusters with HTTP and HTTPS destinations
   - Set `DangerousAcceptAnyServerCertificate=true` for HTTPS cluster
   - Act: Call `config.Update(routes, clusters)`
   - Assert: Configuration accepted (no exception thrown)
   - Assert: Both routes can be accessed (may return BadGateway if backends not running)
6. Create test method `Https_Backend_Accepts_Self_Signed_Certificate()`:
   - Arrange: Configure cluster with HTTPS backend destination
   - Set `HttpClient.DangerousAcceptAnyServerCertificate = true`
   - Set `HttpClient.SslProtocols = SslProtocols.Tls12 | SslProtocols.Tls13`
   - Act: Update configuration and make request
   - Assert: Request processed (no SSL validation error)
   - Assert: YARP doesn't reject self-signed certificates
7. Create test method `Protocol_Specific_Routes_Work_Independently()`:
   - Arrange: Configure 4 routes (2 HTTP backends, 2 HTTPS backends)
   - Act: Make requests to all 4 routes
   - Assert: All routing configurations accepted
   - Assert: No cross-contamination between HTTP and HTTPS routes
8. Create test method `Https_Backend_Requires_Valid_Ssl_Configuration()`:
   - Arrange: Create HTTPS backend without `DangerousAcceptAnyServerCertificate`
   - Act: Try to update configuration
   - Assert: Either configuration succeeds (YARP default) or fails with clear error
   - Document expected behavior in test comment
