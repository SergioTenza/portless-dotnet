    **Create CertInstallCommand.cs:**
    - Inherit from AsyncCommand&lt;CertInstallSettings&gt;
    - Inject ICertificateManager and ICertificateTrustService via constructor
    - Inject ILogger&lt;CertInstallCommand&gt; for logging

    **ExecuteAsync implementation:**
    - Check admin status: if (!WindowsAdminDetector.IsRunningAsAdministrator())
      * Trigger Windows UAC elevation prompt (restart current process with admin privileges using Process.Start with runas verb)
      * If elevation fails or user cancels: display "[yellow]Administrator privileges required. UAC prompt was declined or failed.[/]" and return 2
      * If elevation succeeds: exit current process (elevated process continues)
    - Load CA certificate: var cert = await _certificateManager.GetCertificateAuthorityAsync()
    - If null: display "[red]Error:[/] No certificate found. Run 'portless proxy start' to generate." and return 3 (exit code per CONTEXT.md: Certificate file missing)
    - Check certificate integrity: if cert is corrupted (try/catch around cert access)
      * Prompt user: "Certificate file is corrupted. Regenerate? (y/N)"
      * If yes: regenerate using _certificateManager, else return 4 (exit code per CONTEXT.md: Certificate file corrupted)
    - Install certificate: var result = await _trustService.InstallCertificateAuthorityAsync(cert)
    - Handle results:
      * result.Success &amp;&amp; !result.AlreadyInstalled: display "[green]✓[/] Certificate installed successfully" and return 0
      * result.AlreadyInstalled: display "[green]✓[/] Certificate already installed" and return 0 (idempotent per CONTEXT.md)
      * result.StoreAccessDenied: display "[red]Error:[/] Certificate store access denied. Check permissions." and return 5 (exit code per CONTEXT.md: Certificate store access denied)
      * result.ErrorMessage != null: display "[red]Error:[/] {result.ErrorMessage}" and return 1
    - Catch exceptions: display "[red]Error:[/] {ex.Message}" and return 1

    **Pattern:** Follow ProxyStatusCommand pattern for Spectre.Console output with Markup for colors

    **ExecuteAsync implementation:**

    - Load certificate metadata: var metadata = await _certificateManager.GetCertificateStatusAsync()
    - If cert == null or metadata == null: display "[yellow]No certificate found.[/]" and return 0 (not an error, just not generated yet)

    - Get trust status: var status = await _trustService.GetTrustStatusAsync(cert.Thumbprint)

    - Display status with color coding per CONTEXT.md:
      * status == TrustStatus.Trusted: "[green]✓ Trusted[/]"
      * status == TrustStatus.NotTrusted: "[red]✗ Not Trusted[/]"
      * status == TrustStatus.ExpiringSoon: "[yellow]⚠ Expiring Soon[/]"
      * status == TrustStatus.Unknown: "[yellow]Unknown (non-Windows platform)[/]"

    - If settings.Verbose == true, display detailed information:
      * "SHA-256: {metadata.Sha256Thumbprint}"
      * "Expires: {metadata.ExpiresAt}" (format as ISO 8601)
      * "Store: Cert:\LocalMachine\Root"
      * "Subject: {cert.Subject}"
      * "Issuer: {cert.Issuer}"
      * "Serial: {cert.SerialNumber}"
      * "Key Size: {cert.PublicKey.Key.Size} bits"

    - If status == TrustStatus.NotTrusted, display installation instructions:
      * "\n[yellow]To install:[/] Run as Administrator: portless cert install"

    - Return 0 on success

    **Pattern:** Follow ProxyStatusCommand pattern for --verbose flag and colored output
