   - Serve static files (wwwroot)
   - Use dynamic PORT from environment variable
6. Create README with setup instructions

**Technical Details:**

`ChatHub.cs`:
```csharp
public class ChatHub : Hub
{
    public async Task SendMessage(string user, string message)
    {
        await Clients.All.SendAsync("ReceiveMessage", user, message);
    }
}
```

`Program.cs` configuration:
```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddSignalR();

var app = builder.Build();

app.UseDefaultFiles();
app.UseStaticFiles();
app.MapHub<ChatHub>("/chathub");

app.Run($"http://0.0.0.0:{Environment.GetEnvironmentVariable("PORT") ?? "5000"}");
```

`wwwroot/index.html`:
- Import SignalR JavaScript client from CDN
- Connect to `/chathub` endpoint
- Handle connection events (connected, disconnected, error)
- Send messages via `connection.invoke("SendMessage", user, message)`
- Receive messages via `connection.on("ReceiveMessage", ...)`

### Task 2: Create Console Client Example

**Goal:** Create .NET console app that demonstrates SignalR client usage

**Steps:**
1. Create new console project: `Examples/SignalRChat.Client/`
2. Add `Microsoft.AspNetCore.SignalR.Client` package reference
3. Create `Program.cs` that:
   - Accepts hub URL as argument or uses default
   - Connects to SignalR hub using HubConnectionBuilder
   - Prompts user for username
   - Allows sending messages
   - Receives and displays broadcast messages
   - Handles connection events and errors
4. Add comments explaining key SignalR client concepts

**Technical Details:**

`Program.cs` structure:
```csharp
var url = args.Length > 0 ? args[0] : "http://localhost:5000/chathub";
var connection = new HubConnectionBuilder()
    .WithUrl(url)
    .Build();

connection.On<string, string>("ReceiveMessage", (user, message) =>
{
    Console.WriteLine($"{user}: {message}");
});

await connection.StartAsync();

Console.WriteLine("Connected. Enter username:");
var username = Console.ReadLine();

Console.WriteLine("Enter messages (empty to quit):");
while (true)
{
    var message = Console.ReadLine();
    if (string.IsNullOrEmpty(message)) break;
    await connection.InvokeAsync("SendMessage", username, message);
}

await connection.StopAsync();
```

### Task 3: Test Chat Through Proxy

**Goal:** Verify SignalR chat works through Portless.NET proxy with multiple clients

**Steps:**
1. Start Portless.NET proxy: `portless proxy start`
2. Register chat hostname: `portless chatsignalr -- dotnet run --project Examples/SignalRChat/`
3. Open browser to `http://chatsignalr.localhost:1355`
4. Verify browser client connects (connection status shows "Connected")
5. Send message from browser, verify it appears in message log
6. Start console client: `dotnet run --project Examples/SignalRChat.Client/ http://chatsignalr.localhost:1355/chathub`
7. Send message from console, verify it appears in browser
8. Send message from browser, verify it appears in console
9. Open second browser window to `http://chatsignalr.localhost:1355`
10. Verify all three clients receive broadcast messages
11. Test connection recovery (stop/start server)

## Risk Analysis

**Risk 1: SignalR falls back to Server-Sent Events instead of WebSocket**
- **Impact:** Slower performance, doesn't demonstrate WebSocket proxy capability
- **Mitigation:** Configure SignalR client to prefer WebSocket transport
- **Verification:** Check browser DevTools Network tab for WebSocket connection (101 Switching Protocols)

**Risk 2: Multiple browser windows share same connection**
- **Impact:** Doesn't demonstrate broadcast properly
- **Mitigation:** Use incognito mode or different browsers for testing
- **Verification:** Each browser window shows unique connection

**Risk 3: Console client URL configuration is confusing**
- **Impact:** Users can't connect console client
- **Mitigation:** Provide clear default URL and document argument usage
- **Verification:** Test with default URL and custom URL
