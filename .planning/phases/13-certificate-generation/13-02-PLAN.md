<task type="auto">
  <name>Implement cross-platform file permission service</name>
  <files>Portless.Core/Services/CertificatePermissionService.cs</files>
  <action>
    Create CertificatePermissionService class implementing ICertificatePermissionService with:

    **CreateSecureDirectoryAsync method:**
    - Use OperatingSystem.IsWindows() for platform detection
    - Windows: Create DirectorySecurity, add FileSystemAccessRule for current user (WindowsIdentity.GetCurrent().User) with FullControl, SetAccessRuleProtection(isProtected: true, preserveInheritance: false), call Directory.CreateDirectory(path, security)
    - Unix (Linux/macOS): Create directory with UnixFileMode.UserRead | UnixFileMode.UserWrite | UnixFileMode.UserExecute (chmod 700)
    - Fallback: Call Directory.CreateDirectory(path) for other platforms

    **SetSecureFilePermissionsAsync method:**
    - Unix: Call File.SetUnixFileMode(path, UnixFileMode.UserRead | UnixFileMode.UserWrite) for chmod 600
    - Windows: Create FileSecurity, add FileSystemAccessRule for current user with FullControl, SetAccessRuleProtection(isProtected: true, preserveInheritance: false), call new FileInfo(path).SetAccessControl(security)
    - Other platforms: No-op (log warning)

    **VerifyFilePermissionsAsync method:**
    - Unix: Use File.GetUnixFileMode() and check for UserRead | UserWrite, return true if both set
    - Windows: Use File.GetAccessControl(), check if current user has Read/Write access, return true if both present
    - Other platforms: Return true (best effort)

    **Key patterns from RESEARCH.md:**
    - Use OperatingSystem.IsWindows() and OperatingSystem.IsLinux() || OperatingSystem.IsMacOS() guards
    - Windows: SetAccessRuleProtection(isProtected: true, preserveInheritance: false) removes inherited permissions
    - Unix: UnixFileMode.UserRead | UnixFileMode.UserWrite = chmod 600 (rw-------)
    - Directory: UnixFileMode.UserRead | UnixFileMode.UserWrite | UnixFileMode.UserExecute = chmod 700 (rwx------)

    Add ILogger<CertificatePermissionService> parameter for logging permission operations.

    Handle PlatformNotSupportedException gracefully with try-catch and logging.
  </action>
  <verify>
    dotnet build Portless.Core/Portless.Core.csproj compiles successfully.
  </verify>
  <done>
    CertificatePermissionService creates directories with chmod 700 (Unix) or ACL restricting to current user (Windows), sets file permissions to chmod 600 (Unix) or ACL (Windows), and verifies permissions are correctly applied.
  </done>
</task>

    - Get state directory path from StateDirectoryProvider
    - Call permissionService.CreateSecureDirectoryAsync(stateDir)
    - Export certificate with cert.Export(X509ContentType.Pkcs12, "") - empty string password per user context
    - Write PFX bytes to ca.pfx path

    **SaveServerCertificateAsync method:**
    - Same pattern as CA but saves to cert.pfx path

    **LoadCertificateAuthorityAsync method:**
    - Return null if ca.pfx doesn't exist (use File.Exists)
    - Load with new X509Certificate2(caPfxPath, "", X509KeyStorageFlags.Exportable)
    - Return null on CryptographicException (corrupted PFX)
    - Verify HasPrivateKey is true, return null if missing

    **LoadServerCertificateAsync method:**
    - Same pattern as CA but loads from cert.pfx path

    **SaveCertificateMetadataAsync method:**
    - Ensure state directory exists via permissionService
    - Serialize CertificateInfo to JSON with JsonSerializer.Serialize(info, new JsonSerializerOptions { WriteIndented = true })
    - Write to cert-info.json path
    - Set file permissions via permissionService

    **LoadCertificateMetadataAsync method:**
    - Return null if cert-info.json doesn't exist
    - Read file content and deserialize with JsonSerializer.Deserialize<CertificateInfo>()
    - Return null on JsonException or IOException

    **CertificateFilesExistAsync method:**
    - Return true if ca.pfx, cert.pfx, and cert-info.json all exist (File.Exists check)

    **Key patterns from RESEARCH.md:**
    - Use X509ContentType.Pkcs12 for PFX export
    - Empty string "" for password (not null - per KB5025823)
    - X509KeyStorageFlags.Exportable when loading
    - Validate HasPrivateKey after load
    - Catch CryptographicException for corrupted PFX files
    - Use System.Text.Json for serialization with WriteIndented for readability
