    Update all three commands to use proper exit codes per CONTEXT.md:

    **Exit code mapping (apply to all commands):**
    - 0: Success (including idempotent operations like already installed/uninstalled)
    - 1: Generic error (certificate missing, exception during operation)
    - 2: Insufficient permissions (not administrator on Windows)
    - 1: Platform not supported (non-Windows for install/uninstall commands)

    **CertInstallCommand.Exit codes:**
    - Return 0: Successfully installed or already installed
    - Return 2: RequiresAdmin (Windows, not running as administrator)
    - Return 1: Certificate missing, platform not supported (non-Windows), or generic error

    **CertStatusCommand.Exit codes:**
    - Return 0: Always return 0 (status check is informational, not an error even if not trusted)
    - No error exit codes for status command

    **CertUninstallCommand.Exit codes:**
    - Return 0: Successfully uninstalled or not found (idempotent)
    - Return 1: Platform not supported (non-Windows) or generic error

    **Pattern:** Match existing CLI command exit code patterns (ProxyStatusCommand returns 0 for success/1 for error)
  </action>
