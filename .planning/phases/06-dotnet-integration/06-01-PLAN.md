<task type="auto">
  <name>Configure PackAsTool settings in Portless.Cli.csproj</name>
  <files>Portless.Cli/Portless.Cli.csproj</files>
  <action>
    Add the following MSBuild properties to Portless.Cli/Portless.Cli.csproj within the PropertyGroup section:

    ```xml
    <!-- Tool packaging configuration -->
    <PackAsTool>true</PackAsTool>
    <ToolCommandName>portless</ToolCommandName>
    <Version>1.0.0</Version>
    <Authors>Portless.NET Contributors</Authors>
    <Description>Stable .localhost URLs for local .NET development</Description>

    <!-- Package metadata -->
    <PackageId>Portless.NET.Tool</PackageId>
    <PackageTags>dotnet-tool;proxy;localhost;development;portless</PackageTags>
    <PackageProjectUrl>https://github.com/portless/portless-dotnet</PackageProjectUrl>
    <RepositoryUrl>https://github.com/portless/portless-dotnet</RepositoryUrl>
    <RepositoryType>git</RepositoryType>

    <!-- Native AOT for single executable (test compatibility with Spectre.Console.Cli) -->
    <PublishAot>true</PublishAot>
    ```

    CRITICAL: Do NOT add `PrivateAssets="all"` to any PackageReference or ProjectReference entries, as this would exclude dependencies from the tool package. All Spectre.Console, Microsoft.Extensions.*, and project references must be included in the package.

    This configuration follows the .NET 10 PackAsTool pattern documented in RESEARCH.md, enabling the tool to be packaged as a NuGet package and installed globally via `dotnet tool install --global Portless.NET.Tool`.
  </action>
  <verify>
    Run `dotnet build Portless.Cli/Portless.Cli.csproj` and verify:
    1. Build completes with 0 errors, 0 warnings
    2. Output shows "ToolCommandName" and "PackAsTool" properties are recognized
    3. No trimming or AOT errors (indicating Spectre.Console.Cli compatibility)
  </verify>
  <done>
    Portless.Cli.csproj contains all required PackAsTool properties, tool builds successfully without AOT errors, and all dependencies are configured for packaging (no PrivateAssets exclusions)
  </done>
</task>
