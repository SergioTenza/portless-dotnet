    Create InMemoryConfigProvider.cs in Portless.Proxy project:
    1. Implement IProxyConfigProvider interface from YARP
    2. Use volatile InMemoryConfig field for thread-safe config swapping
    3. Create constructor accepting initial routes/clusters (optional, defaults to empty)
    4. Implement GetConfig() returning current _config
    5. Implement Update(IReadOnlyList<RouteConfig>, IReadOnlyList<ClusterConfig>) method:
       - Store oldConfig reference
       - Create new InMemoryConfig with new routes/clusters
       - Assign to _config field
       - Call oldConfig.SignalChange() to trigger YARP reload
    6. Register as singleton in DI: builder.Services.AddSingleton<InMemoryConfigProvider>()

    Implementation pattern from research:
    ```csharp
    public class InMemoryConfigProvider : IProxyConfigProvider
    {
        private volatile InMemoryConfig _config;

        public InMemoryConfigProvider()
        {
            _config = new InMemoryConfig(Array.Empty<RouteConfig>(), Array.Empty<ClusterConfig>());
        }

        public IProxyConfig GetConfig() => _config;

        public void Update(IReadOnlyList<RouteConfig> routes, IReadOnlyList<ClusterConfig> clusters)
        {
            var oldConfig = _config;
            _config = new InMemoryConfig(routes, clusters);
            oldConfig.SignalChange();
        }
    }
    ```

    DO NOT add file I/O or persistence - that's Phase 2. Keep config in-memory only.
