## Milestone Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| .NET native APIs over BouncyCastle | .NET 10 has full certificate support, no external deps | ✅ Good - Simple, built-in APIs |
| PFX with password over PEM | Secure private key storage, cross-platform compatible | ✅ Good - Secure file format |
| Three-file storage strategy | Separate CA, wildcard cert, metadata | ✅ Good - Clean separation |
| Windows-only automated trust | macOS/Linux complex, defer to v1.3+ | ✅ Good - Focus on Windows |
| LocalMachine Root store | System-wide trust, requires admin | ✅ Good - Proper security model |
| Manual macOS/Linux trust | Automated trust complex (keychain, distribution) | ✅ Good - Honest limitations |
| 30-day expiration warning | Industry standard para certificate renewal | ✅ Good - Proactive notifications |
| Idempotent operations | Install/uninstall succeed on duplicate/no-op | ✅ Good - Better UX |
| Exit codes follow CONTEXT.md spec | Consistent error reporting | ✅ Good - Predictable behavior |
| Deferred HTTPS proxy integration | Certificate management complete, proxy can be added later | ⚠️ TBD - Users may expect HTTPS proxy |

## Known Limitations

### By Design (Deferred)
- **HTTPS Proxy:** Certificate management complete, but HTTPS proxy not implemented
- **Background Monitoring:** Certificate lifecycle monitoring not implemented
- **Integration Tests:** Automated tests not written (manual testing performed)
- **Documentation:** User guides not created (help text available)

### Platform Limitations
- **Windows Only:** Automated trust installation works only on Windows 10+
- **Admin Required:** Certificate trust operations require administrator privileges
- **Manual macOS/Linux:** Users must manually install CA certificate on non-Windows platforms

### Certificate Limitations
- **Self-Signed:** Certificates are self-signed (not suitable for production)
- **5-Year Validity:** Fixed expiration (not configurable)
- **Single CA:** Only one CA certificate supported

## Migration Path

**For Users Upgrading from v1.1:**
1. Install v1.2 via `dotnet tool install --global portless.dotnet`
2. Run `portless cert install` (Windows only, requires admin)
3. Run `portless cert status` to verify trust
4. Run `portless cert renew` if expiring within 30 days

**Breaking Changes:** None

**Rollback:** Uninstall v1.2, install v1.1 (certificates in `~/.portless/` preserved)
