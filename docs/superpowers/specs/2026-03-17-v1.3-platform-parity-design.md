# Design Specification: v1.3 Platform Parity - Certificate Trust Automation

**Date:** 2026-03-17
**Author:** Claude Code
**Status:** Draft
**Milestone:** v1.3 Platform Expansion

## 1. Overview

This specification defines a multi-platform certificate trust automation system for Portless.NET v1.3, achieving feature parity between Windows (v1.2) and macOS/Linux platforms (v1.3).

### Problem Statement

**Current State (v1.2):**
- **Windows 10+**: Automated trust installation with one command ✅
- **macOS 12+**: Manual installation required (345 lines of documentation) ⚠️
- **Linux**: Manual installation required (345 lines of documentation) ⚠️

**User Impact:**
- macOS/Linux users must execute complex manual commands
- 345 lines of troubleshooting documentation needed
- Inconsistent experience across platforms
- Higher barrier to entry for non-Windows users

### Solution

Implement platform-specific certificate trust automation for macOS and Linux, matching the Windows experience from v1.2.

**Target State (v1.3):**
- **Windows 10+**: Automated trust installation ✅ (unchanged)
- **macOS 12+**: Automated trust installation ✅ (NEW)
- **Linux**: Automated trust installation ✅ (NEW)

**Benefits:**
- Consistent user experience across all platforms
- Reduce documentation from 345 lines to ~50 lines (troubleshooting only)
- Lower barrier to entry for macOS/Linux developers
- Professional tool matching .NET, Docker, and other dev tools

## 2. Architecture

### Multi-Platform Specialization Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    CLI Commands Layer                        │
│  CertInstallCommand (Windows)                                │
│  CertInstallCommand (macOS)  - NEW                           │
│  CertInstallCommand (Linux)  - NEW                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              Abstraction Layer (ICertificateTrustService)     │
│  Detects platform → selects implementation                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
        ┌──────────────────────┼──────────────────────┐
        ↓                      ↓                      ↓
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Windows    │    │    macOS     │    │    Linux     │
│ Implementation│    │ Implementation│    │ Implementation│
│ (v1.2 exists)│    │   (NEW v1.3) │    │   (NEW v1.3) │
└──────────────┘    └──────────────┘    └──────────────┘
    X509Store          Security            Distribution-
   -System Root      Framework          Specific Stores
```

### Key Principles

1. **Platform Detection**: `RuntimeInformation.IsOSPlatform()` for implementation selection
2. **Platform Guards**: `[SupportedOSPlatform("macos10.12")]`, `[SupportedOSPlatform("linux")]`
3. **Consistent Interface**: Same `ICertificateTrustService` for all platforms
4. **Graceful Degradation**: Manual instructions if automation fails

## 3. Components

### Component 1: CertificateTrustServiceMacOS (NEW)

**Responsibilities:**
- Manipulate System Keychain via Security framework
- Detect if certificate is already trusted
- Add/remove CA certificates with admin rights
- Detect if Keychain requires unlock

```csharp
[SupportedOSPlatform("macos10.12")]
public class CertificateTrustServiceMacOS : ICertificateTrustService
{
    // P/Invoke declarations for Security framework
    [DllImport("/System/Library/Frameworks/Security.framework/Security")]
    private static extern int SecTrustCopyAnchorCertificates(ref IntPtr certs);

    public Task<TrustInstallResult> InstallTrustAsync(X509Certificate2 cert, CancellationToken ct)
    public Task<TrustInstallResult> RemoveTrustAsync(string thumbprint, CancellationToken ct)
    public Task<TrustStatus> GetTrustStatusAsync(string thumbprint, CancellationToken ct)
}
```

**Platform-specific considerations:**
- Requires `sudo` privileges for System Keychain access
- Keychain may be locked (needs unlock or user interaction)
- System Integrity Protection (SIP) may block access
- P/Invoke required for Security framework APIs

### Component 2: CertificateTrustServiceLinux (NEW)

**Responsibilities:**
- Detect Linux distribution (read `/etc/os-release`)
- Copy certificate to distribution-specific path
- Execute certificate update command
- Detect permissions (requires `sudo`)

```csharp
[SupportedOSPlatform("linux")]
public class CertificateTrustServiceLinux : ICertificateTrustService
{
    // Distribution-specific paths
    private static readonly Dictionary<string, LinuxDistroInfo> DistroPaths = new()
    {
        ["ubuntu"] = new("/usr/local/share/ca-certificates/", "update-ca-certificates"),
        ["debian"] = new("/usr/local/share/ca-certificates/", "update-ca-certificates"),
        ["fedora"] = new("/etc/pki/ca-trust/source/anchors/", "update-ca-trust"),
        ["rhel"] = new("/etc/pki/ca-trust/source/anchors/", "update-ca-trust"),
        ["arch"] = new("/etc/ca-certificates/trust-source/anchors/", "trust anchor")
    };

    public Task<TrustInstallResult> InstallTrustAsync(X509Certificate2 cert, CancellationToken ct)
    public Task<TrustInstallResult> RemoveTrustAsync(string thumbprint, CancellationToken ct)
    public Task<TrustStatus> GetTrustStatusAsync(string thumbprint, CancellationToken ct)
}
```

**Platform-specific considerations:**
- Requires `root` privileges for system certificate stores
- Different distributions use different paths and commands
- May require `update-ca-certificates --fresh` on some systems
- Must handle various Linux distributions (Ubuntu, Debian, Fedora, RHEL, Arch)

### Component 3: PlatformDetectorService (NEW)

**Responsibilities:**
- Detect current OS (Windows/macOS/Linux)
- Detect specific Linux distribution
- Verify if user has admin permissions
- Generate elevation commands (`sudo`, `sudo -E`, etc.)

```csharp
public class PlatformDetectorService : IPlatformDetectorService
{
    public PlatformInfo GetPlatformInfo()
    public bool IsAdminUser()
    public string GetAdminElevationCommand()
}
```

### Component 4: CertificateTrustServiceFactory (UPDATED)

**Responsibilities:**
- Factory pattern for creating correct implementation
- Detect platform at runtime
- Provide fallback implementation if not supported

```csharp
public class CertificateTrustServiceFactory : ICertificateTrustServiceFactory
{
    public ICertificateTrustService CreateTrustService()
    {
        if (RuntimeInformation.IsOSPlatform(OSPlatform.Windows))
            return new CertificateTrustServiceWindows(); // v1.2
        if (RuntimeInformation.IsOSPlatform(OSPlatform.OSX))
            return new CertificateTrustServiceMacOS(); // NEW v1.3
        if (RuntimeInformation.IsOSPlatform(OSPlatform.Linux))
            return new CertificateTrustServiceLinux(); // NEW v1.3

        throw new PlatformNotSupportedException("Unsupported platform");
    }
}
```

## 4. Data Flow

### General Flow (Platform-Agnostic)

```
Usuario ejecuta: portless cert install
        ↓
CertInstallCommand detecta plataforma
        ↓
CertificateTrustServiceFactory crea servicio específico
        ↓
ICertificateTrustService.InstallTrustAsync()
        ↓
Resultado: TrustInstallResult (Success/Failed + detalles)
```

### macOS Flow (v1.3 NEW)

```
CertInstallCommand macos
    ↓
Detectar: ¿Usuario es admin?
    ↓
    NO: Mostrar instrucciones con 'sudo'
    SI: Continuar
    ↓
CertificateTrustServiceMacOS.InstallTrustAsync()
    ↓
    1. Convertir PFX → PEM (openssl)
    2. Security framework: SecTrustCopyAnchorCertificates()
    3. Buscar certificado por thumbprint
    ↓
    ¿Ya existe en System Keychain?
    ↓
    SI: Return TrustInstallResult.AlreadyInstalled
    NO: Continuar
    ↓
    4. Security framework: SecItemImport()
    5. Security framework: SecTrustSettingsSetTrust()
    6. Verificar: SecItemCopyMatching()
    ↓
Return TrustInstallResult.Success
```

### Linux Flow (v1.3 NEW)

```
CertInstallCommand linux
    ↓
Detectar distribución (leer /etc/os-release)
    ↓
    Ubuntu/Debian: /usr/local/share/ca-certificates/
    Fedora/RHEL: /etc/pki/ca-trust/source/anchors/
    Arch: /etc/ca-certificates/trust-source/anchors/
    ↓
Detectar: ¿Usuario es root?
    ↓
    NO: Mostrar instrucciones con 'sudo'
    SI: Continuar
    ↓
CertificateTrustServiceLinux.InstallTrustAsync()
    ↓
    1. Convertir PFX → CRT (openssl)
    2. Copiar a path específico de distribución
    3. Ejecutar comando de actualización
        - Ubuntu/Debian: update-ca-certificates
        - Fedora/RHEL: update-ca-trust
        - Arch: trust anchor
    ↓
Return TrustInstallResult.Success
```

## 5. Error Handling

### macOS Edge Cases

**1. Keychain Locked**
```bash
portless cert install
⚠️  System Keychain is locked
💡 Unlock Keychain:
   sudo security unlock-keychain /Library/Keychains/System.keychain
```

**2. SIP (System Integrity Protection) Blocks**
```bash
portless cert install
❌ Installation failed: System Integrity Protection blocks access to System Keychain
💡 Manual installation:
   sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ~/.portless/ca.pfx
```

**3. No Security Framework Available**
```bash
portless cert install
❌ Security framework not found (macOS < 10.12)
💡 Manual installation:
   sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ~/.portless/ca.pfx
```

### Linux Edge Cases

**1. Not Root**
```bash
portless cert install
❌ Permission denied: Writing to /usr/local/share/ca-certificates/
💡 Run with sudo: sudo portless cert install
```

**2. Distribution Not Detected**
```bash
portless cert install
❌ Unsupported Linux distribution: detected "unknown"
💡 Manual installation required for your distribution
   See: https://github.com/SergioTenza/portless-dotnet/blob/main/docs/certificate-troubleshooting-macos-linux.md
```

**3. Certificate Command Failed**
```bash
portless cert install
✅ Certificate copied to /usr/local/share/ca-certificates/portless-ca.crt
❌ Command failed: update-ca-certificates exited with code 1
💡 Try running manually: sudo update-ca-certificates --fresh
```

### Cross-Platform Edge Cases

**1. Certificate Corrupted**
```bash
portless cert install
❌ Certificate file corrupted: ~/.portless/ca.pfx
💡 Regenerate certificate: portless cert renew --force
```

**2. Certificate Already Installed**
```bash
portless cert install
ℹ️  Certificate already installed (thumbprint: ABC123...)
No action needed. Your *.localhost certificates are already trusted.
```

### Fallback Strategy

If automation fails, provide platform-specific manual instructions:

```csharp
public class ManualInstallationInstructions
{
    public static string GenerateForLinux(LinuxDistro distro)
    {
        return distro switch
        {
            LinuxDistro.Ubuntu => """
                # Manual installation for Ubuntu/Debian:
                sudo cp ~/.portless/ca.pfx /usr/local/share/ca-certificates/portless-ca.crt
                sudo update-ca-certificates --fresh
                """,
            LinuxDistro.Fedora => """
                # Manual installation for Fedora/RHEL:
                sudo cp ~/.portless/ca.pfx /etc/pki/ca-trust/source/anchors/portless-ca.crt
                sudo update-ca-trust
                """,
            _ => "# Manual installation: See docs/certificate-troubleshooting-macos-linux.md"
        };
    }
}
```

## 6. Testing Strategy

### macOS Testing

**Unit Tests (Platform-Specific)**
```csharp
[Fact]
[SupportedOSPlatform("macos10.12")]
public async Task InstallTrust_AddsToSystemKeychain()
{
    // Arrange
    var service = new CertificateTrustServiceMacOS();
    var cert = await CreateTestCertificate();

    // Act
    var result = await service.InstallTrustAsync(cert, CancellationToken.None);

    // Assert
    Assert.Equal(TrustInstallStatus.Success, result.Status);
    Assert.True(await service.IsCertificateTrustedAsync(cert.Thumbprint));
}
```

**Integration Tests (Require macOS Real)**
- Verify certificate appears in Keychain Access
- Verify browser accepts certificate without warnings
- Verify `portless cert status` shows "Trusted"

## 9. Risks and Mitigations

### Risk 1: macOS Security Framework Complexity

**Probability:** HIGH | **Impact:** MEDIUM

P/Invoke with Security framework may be more complex than anticipated.

**Mitigation:**
- Research phase before implementation
- Consider using `openssl` command-line tool as fallback
- Comprehensive error handling with clear user messages

### Risk 2: Linux Distribution Fragmentation

**Probability:** MEDIUM | **Impact:** MEDIUM

Many Linux distributions exist; cannot support all.

**Mitigation:**
- Focus on top 5 distributions (Ubuntu, Debian, Fedora, RHEL, Arch)
- Provide clear manual installation instructions for unsupported distros
- Detect distribution and provide specific error messages

### Risk 3: Lack of Physical Hardware for Testing

**Probability:** MEDIUM | **Impact:** HIGH

Cannot test on real macOS or various Linux distributions.

**Mitigation:**
- Use Docker containers for Linux testing
- Use VMs for macOS testing (if legal/licensed)
- Rely on community testing via beta releases
- Comprehensive unit tests with mocked dependencies

### Risk 4: Breaking Changes to v1.2

**Probability:** LOW | **Impact:** HIGH

Changes to `ICertificateTrustService` or factory could break Windows.

**Mitigation:**
- Windows implementation remains unchanged
- Add new services without modifying existing ones
- Comprehensive regression testing on Windows
- Platform guards prevent wrong code from running
