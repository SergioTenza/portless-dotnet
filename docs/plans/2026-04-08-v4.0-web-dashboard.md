# Portless.NET v4.0 - Web Dashboard Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Add a real-time web dashboard served by the proxy at `http://proxy.localhost:1355/_dashboard/` for monitoring routes, requests, metrics, and plugins.

**Architecture:** The dashboard is embedded as static HTML/CSS/JS files in `Portless.Proxy/wwwroot/dashboard/`. New API endpoints provide data. Server-Sent Events (SSE) push live updates. No build step, no Node.js dependency — vanilla JS + Chart.js from CDN.

**Tech Stack:** ASP.NET Core static files, Minimal API, SSE, vanilla HTML/CSS/JS, Chart.js (CDN)

---

## Phase 1: Infrastructure (Backend)

### Task 1: Enable static files and dashboard routing

**Objective:** Configure the proxy to serve static files from wwwroot and register the `/_dashboard/` path.

**Files:**
- Modify: `Portless.Proxy/Program.cs` (add UseStaticFiles, MapStaticAssets)
- Create: `Portless.Proxy/wwwroot/dashboard/index.html` (placeholder)

**Step 1:** Add static files middleware to Program.cs, before the proxy pipeline. Must exclude `/_dashboard/` from YARP routing.

```csharp
// After app.UseWebSockets(); and before app.MapReverseProxy():
app.UseStaticFiles(new StaticFileOptions
{
    ServeUnknownFileTypes = true,
    DefaultContentType = "text/html"
});
```

**Step 2:** Create a minimal `index.html` placeholder at `wwwroot/dashboard/index.html` with "Portless Dashboard v4.0" title.

**Step 3:** Verify that `http://proxy.localhost:1355/_dashboard/` serves the HTML file.

**Step 4:** Commit.

### Task 2: Add dashboard API endpoints

**Objective:** Create new API endpoints that the dashboard frontend will consume.

**Files:**
- Create: `Portless.Proxy/DashboardApiEndpoints.cs`
- Modify: `Portless.Proxy/Program.cs` (register endpoints)

**Endpoints to add:**

1. `GET /api/v1/dashboard/summary` — Aggregated overview (route count, uptime, total requests, error rate, requests/min)
2. `GET /api/v1/dashboard/routes` — Routes with health status (is backend reachable?)
3. `GET /api/v1/dashboard/metrics/history` — Last N minutes of request metrics (for charts)
4. `GET /api/v1/dashboard/events` — SSE endpoint for live events (new requests, route changes, errors)

**Step 1:** Create `DashboardApiEndpoints.cs` with the `MapDashboardApi()` extension method.

**Step 2:** Register in Program.cs: `app.MapDashboardApi(...)`.

**Step 3:** Write unit tests for the new endpoints.

**Step 4:** Commit.

---

### Task 3: Add route health check service

**Objective:** Periodically check if backend services are reachable and expose health status.

**Files:**
- Create: `Portless.Core/Services/IRouteHealthChecker.cs` (interface)
- Create: `Portless.Core/Services/RouteHealthChecker.cs` (implementation)
- Modify: `Portless.Proxy/Program.cs` (register as hosted service)

**Implementation:**
- Background service that checks each route's backend every 10 seconds
- HTTP health probe (HEAD request with 2s timeout)
- Stores health status: Healthy, Unhealthy, Unknown
- Exposes via DI for the dashboard API

**Step 1:** Create interface and implementation.

**Step 2:** Register as hosted service in Program.cs.

**Step 3:** Write tests.

**Step 4:** Commit.

---

### Task 4: Add SSE live events endpoint

**Objective:** Push real-time events to the dashboard via Server-Sent Events.

**Files:**
- Modify: `Portless.Proxy/DashboardApiEndpoints.cs`
- Create: `Portless.Core/Services/IEventBus.cs` (simple pub/sub interface)
- Create: `Portless.Core/Services/EventBus.cs` (implementation using Channel)

**Events:**
- `request.completed` — After each proxied request (debounced, batch every 500ms)
- `route.added` / `route.removed` — Route changes
- `health.changed` — Backend health status change

**Step 1:** Create EventBus with Channel<T> for async pub/sub.

**Step 2:** Add SSE endpoint that subscribes to the event bus.

**Step 3:** Hook EventBus into RequestLoggingMiddleware, PortlessApiEndpoints, and RouteHealthChecker.

**Step 4:** Write tests.

**Step 5:** Commit.
